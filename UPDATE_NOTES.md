# 从原版到当前版本的更新说明

本文记录本次在原始 `3d-resource-space` 项目基础上的功能扩展、工程改动和验证结果，便于提交 PR 时说明改动范围。

## 背景

原版项目已经具备一套基础的 3D 资源空间可视化能力：

- 可以创建中心点，中心点作为已知坐标锚点。
- 可以创建资源球，资源球通过关联两个及以上中心点并配置权重，使用加权平均计算坐标。
- 可以在 Three.js 场景中展示中心点、资源球和关联线。
- 可以做投影查询：从一个资源球朝另一个资源球方向发出投影圆柱，筛选路径上相交或被包含的资源球。

但原版还停留在“已知资源球可视化”和“投影查询”阶段，没有把“问题资源球如何被求解出来”闭环实现。也就是说，项目里已经有了找参考球的能力，但还没有把这些参考球反过来用于推断未知资源球的位置。

## 本次更新目标

本次更新把项目从“3D 可视化 demo”推进到“空间推理实验台”的 MVP：

1. 增加问题资源球求解器。
2. 用最优候选点的前 5% 拟合置信度椭球。
3. 支持多个方向目标球联合约束求解。
4. 支持把求解结果保存回数据库，成为新的资源球。
5. 在前端 3D 场景中展示求解过程和结果。
6. 增加测试覆盖，保证核心几何计算稳定。

## 后端更新

### 新增 `app/solver.py`

新增问题资源球求解器，核心思想如下：

- 问题球只锚定到一个中心点，因此只能确定“距锚点的模长”，候选位置落在以锚点为球心、模长为半径的球面上。
- 使用 Fibonacci sphere sampling 在球面上均匀采样候选点。
- 对每个候选点，向一个或多个方向目标球发射射线。
- 计算这些射线到路径参考球中心的距离平方和，作为 fitness。
- fitness 越低，说明该候选点越符合“从问题球朝目标球方向投影时，应经过这些参考资源球”的约束。
- 返回最优候选点、所有候选点、方向向量和置信度椭球。

主要返回结构包括：

- `best`：最优解位置、fitness、方向向量。
- `candidates`：全部候选点及其 fitness，用于前端点云热力图。
- `confidence`：根据 fitness 最好的前 5% 候选点拟合的轴对齐置信椭球。

### 新增 `/api/solve`

新增求解接口：

```http
POST /api/solve
```

请求支持：

- `anchor_center_id`：问题球锚定的中心点。
- `radius`：问题球到锚点的距离，也就是模长。
- `target_sphere_ids`：一个或多个方向目标球，用于多目标联立求解。
- `target_sphere_id`：兼容旧的单目标字段。
- `reference_sphere_ids`：路径参考球集合。
- `samples`：球面候选采样数量。
- `save_as_name`：可选；如果填写，则将最优解保存为数据库中的资源球。

### 多目标联立

原先 MVP 只考虑单个方向目标球。本次更新后，求解器可以同时接收多个目标球：

- 每个候选点会分别朝多个目标球生成方向射线。
- 每条射线都会对所有参考球计算距离误差。
- 多条射线的误差共同累加为候选点的总 fitness。

这样可以把多个方向约束交叉起来，减少单条方向线带来的几何退化问题。

### 置信度椭球

新增置信度估计：

- 按 fitness 从低到高排序所有候选点。
- 取前 5% 的最佳候选点。
- 计算这些点的中心。
- 用各坐标轴上的最大偏移估计轴对齐椭球半径。

这个椭球不是严格统计意义上的协方差椭球，而是一个直观、稳定、便于 Three.js 渲染的误差范围表达，用来展示“当前解大概稳定在哪个空间范围内”。

### 结果回写

当 `/api/solve` 请求包含 `save_as_name` 时：

- 后端会把最优解保存到 `resource_spheres`。
- 保存的资源球 `radius` 使用本次求解输入的模长。
- 坐标使用最优解坐标。
- 自动写入一条与锚点的关系，权重为 `1.0`。
- 新增字段 `is_solved=1` 标记该资源球来自求解器。

### 数据库迁移

`resource_spheres` 新增字段：

```sql
is_solved INTEGER NOT NULL DEFAULT 0
```

同时在 `init_db()` 中加入轻量迁移逻辑：如果旧库没有 `is_solved` 字段，会自动 `ALTER TABLE` 补上，不需要手动删除旧数据库。

## 前端更新

### 新增“求解问题球”面板

在右侧控制面板新增求解区域：

- 锚定中心点选择。
- 模长输入。
- 方向参考目标球多选。
- 路径参考球多选。
- 采样点数输入。
- 保存为资源球名称输入。
- 开始求解按钮。
- 清除求解结果按钮。

### 3D 可视化新增内容

点击“开始求解”后，Three.js 场景会展示：

- 蓝紫色线框球面：问题球候选位置所在球面。
- 候选点云：所有采样候选点，按 fitness 着色。
- 黄色球：最优解位置。
- 黄色线：从最优解指向每个方向目标球的射线。
- 白色线框椭球：fitness 最好的前 5% 候选点拟合出的置信度椭球。

### 保存后自动刷新

如果填写了保存名称，求解成功后：

- 后端保存新的资源球。
- 前端清空保存名称输入框。
- 自动重新加载全部中心点和资源球。
- 新保存的求解资源球会出现在场景和元素列表中。

## 测试更新

新增 `tests/test_solver.py`，覆盖：

- Fibonacci 球面采样数量。
- 采样点是否落在单位球面上。
- 点到射线距离计算。
- 点在射线反方向时的距离退化逻辑。
- 无参考球时所有候选点 fitness 为 0。
- 候选点是否落在锚点球面上。
- 单目标参考约束能否选出预期方向。
- 多目标联立是否返回多个方向向量。
- 非法半径和非法采样数校验。

当前测试结果：

```text
21 passed
```

## 与原版的主要差异

原版：

- 中心点和资源球 CRUD。
- 加权平均计算资源球坐标。
- 3D 展示中心点、资源球、关联线。
- 投影查询只能返回路径上的参考资源球列表。

当前版本：

- 增加问题资源球求解器。
- 投影参考可以参与反推未知资源球位置。
- 支持多个方向目标球联合求解。
- 返回最优解、候选点云、fitness 和置信度椭球。
- 支持将求解结果保存为新的资源球。
- 前端可以直接操作求解流程并在 3D 场景中查看推理过程。
- 新增数据库字段区分普通资源球和求解生成的资源球。
- 新增核心几何计算测试。

## 使用建议

一个典型流程：

1. 创建多个中心点作为空间锚点。
2. 创建若干已知资源球。
3. 在“求解问题球”面板中选择一个锚点。
4. 输入问题球模长。
5. 选择一个或多个方向目标球。
6. 选择路径上的参考球。
7. 点击“开始求解”。
8. 查看候选点云、最优解和置信度椭球。
9. 如结果合理，填写名称并保存为新的资源球。

## 后续可继续改进的方向

- 把置信度椭球从轴对齐近似升级为 PCA / 协方差椭球。
- 给不同参考球加入权重，让某些参考球对 fitness 影响更大。
- 支持将求解记录持久化，便于复盘每次求解过程。
- 为 `/api/solve` 增加接口级集成测试。
- 改进前端多选交互，增加搜索和分组。
- 增加求解结果的导入导出。
